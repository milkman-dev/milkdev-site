getting AWS certified

03 September 2025

i passed the AWS cloud practitioner exam last week, the foundational one, took about six weeks of studying alongside everything else and i'm glad i did it

the reason i wanted it isn't because i'm planning to become a cloud engineer, it's because i think understanding infrastructure is one of those foundations that will matter more as everything else gets automated, not less

here's the thing about AI replacing code, even if it eventually writes everything, the code still needs to run somewhere, it needs a server, a database, a network, a way to scale when traffic spikes, a way to recover when something fails, none of that disappears because the code was generated instead of written by hand

the cloud layer is the interesting part because it's genuinely unclear how much of it AI can fully replace, you can imagine it picking the right instance type or writing the terraform, but the decisions about architecture, about tradeoffs between cost and performance and reliability, those feel like they still need a human who understands the actual context of what you're building and why

maybe i'm wrong about that and in a few years AI handles all of it too, but i don't think so, i think cloud infrastructure is one of those areas where the consequences of getting it wrong are significant enough that humans stay in the loop for a long time

i'd recommend the cloud practitioner cert to anyone who works in software or product or anything adjacent, it's not about becoming a devops specialist, it's about understanding the environment your software actually lives in, and that understanding is useful regardless of what else changes
